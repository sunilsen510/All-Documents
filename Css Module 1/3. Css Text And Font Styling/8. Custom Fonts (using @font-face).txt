1. Custom Fonts क्या हैं?

Definition:

Custom Fonts वह fonts हैं जिन्हें आप खुद host या import करते हैं, ताकि आप default या Google Fonts से अलग font अपने web page में use कर सकें।

यह @font-face CSS rule के जरिए implement होते हैं।

Advantages:

Unique typography / branding

Cross-browser support

Control over font weight, style, format

Works offline if font hosted locally

2. @font-face Syntax
@font-face {
  font-family: 'MyCustomFont';  /* नाम assign करें */
  src: url('fonts/MyCustomFont.woff2') format('woff2'),
       url('fonts/MyCustomFont.woff') format('woff');
  font-weight: normal;
  font-style: normal;
}


Parameters Explained:

font-family: इस font को CSS में refer करने का नाम

src: Font file path + format

font-weight / font-style: Font के weight और style define करना

3. Font Formats

Tip: Usually WOFF2 + WOFF cover most browsers

4. Example – Custom Font Integration

Folder Structure:

project/
│
├── index.html
└── css/
    └── style.css
└── fonts/
    ├── MyCustomFont.woff2
    └── MyCustomFont.woff


CSS (style.css):

@font-face {
  font-family: 'MyCustomFont';
  src: url('../fonts/MyCustomFont.woff2') format('woff2'),
       url('../fonts/MyCustomFont.woff') format('woff');
  font-weight: normal;
  font-style: normal;
}

body {
  font-family: 'MyCustomFont', Arial, sans-serif;
}

h1 {
  font-weight: bold;
}

p {
  font-weight: normal;
}

<!DOCTYPE html>
<html lang="hi">
<head>

<link rel="stylesheet" href="css/style.css">
<title>Custom Font Example</title>
</head>
<body>
<h1>यह हेडिंग MyCustomFont में है</h1>

</body>
</html>

2. TTF (TrueType Font)

Definition: Desktop fonts के लिए standard format।

Features:

Vector-based → scalable without losing quality

Cross-platform → Windows, Mac, Linux support

Use-case:

Desktop applications, graphic design

Web fonts, लेकिन size बड़ा होने की वजह से performance slow

Example: MyFont.ttf

Pros: Easy to use, widely supported
Cons: Larger file size, not optimal for web

3. OTF (OpenType Font)

Definition: TTF का upgraded version with advanced features

Features:

Ligatures, alternate glyphs, advanced typography

Cross-platform support

Use-case: Professional typography, print, design-heavy websites

Example: MyFont.otf

Pros: Advanced typographic features
Cons: Large file, browser performance slower

4. WOFF (Web Open Font Format)

Definition: TTF/OTF का compressed version specially web के लिए

Features:

Smaller file size → fast loading

Metadata support for web

Use-case: Websites, cross-browser support

Example: MyFont.woff

Pros: Optimized for web, good browser support
Cons: Slightly bigger than WOFF2

5. WOFF2

Definition: Latest version of WOFF, highly compressed

Features:

Best performance and smallest file size

Modern browsers support only

Use-case: Websites, modern web apps

Example: MyFont.woff2

Pros: Fast load, smaller size → better performance
Cons: Old browsers (IE < 11) support नहीं करता

6. EOT (Embedded OpenType)

Definition: Legacy IE font format

Use-case: Old Internet Explorer compatibility

Example: MyFont.eot

Note: अब ज्यादातर use नहीं होता

7. SVG Fonts

Definition: Font vector in SVG format

Use-case: Old iOS devices (iOS 4.1–4.3)

Note: Rarely used now

8. Practical Example – Multiple Font Formats (Cross-Browser)
@font-face {
  font-family: 'MyCustomFont';
  src: url('fonts/MyCustomFont.woff2') format('woff2'),
       url('fonts/MyCustomFont.woff') format('woff'),
       url('fonts/MyCustomFont.ttf') format('truetype');
  font-weight: normal;
  font-style: normal;
}


Explanation:

Browser → पहले WOFF2 load करेगा (modern, fast)

Browser → फिर WOFF load करेगा (older modern)

Browser → फिर TTF load करेगा (legacy support)

हमेशा fallback font-family भी specify करें:

body {
  font-family: 'MyCustomFont', Arial, sans-serif;
}

| -------------------- | ------------------ |

10. Key Points / Tips

WOFF2 → best choice for performance

Always provide multiple formats → cross-browser support

TTF / OTF → larger, use only if necessary

SVG / EOT → rare, legacy support

Always specify fallback fonts for safety
